Lý thuyết React căn bản
Modified 2023-08-01

Component: Chia nhỏ UI, tái sử dụng, xử lý từng phần riêng biệt (Class component, functional component)
Props: Component cũng giống function trong JS, nên có thể nhận các tham chiếu đầu vào
State: Kho lưu trữ dữ liệu trong React
LifeCycle: Vòng đời (Mounting, Updating, Unmounting)
Handling Events: CamelCase, sử dụng hàm thay vì chuỗi
Conditional rendering: Render theo điều kiện
List, key: Key giúp React định danh các phần tử trong list, giúp thêm sửa xóa dễ hơn
Context: Truyền dữ liệu qua cây thành phần mà không cần truyền props thủ công
Error Boundaries: Giống try catch
useRef: Truy cập vào DOM nodes hoặc React elements trong phương thức render
Higher-order component: Hàm lấy về 1 element và trả 1 element mới
Fragment: Gom gói các element con để return
Portals: Giúp style của component không bị ảnh hưởng bởi style của component cha mà chỉ bị ảnh hưởng bởi global style
Strict Mode: Chế độ nghiêm ngặt cho các phần tử con nằm trong nó
Hooks: Cho phép sử dụng state ngay trong functional component
useState: Định nghĩa state variables
useEffect: Quản lý vòng đời components, render khi param thứ 2 thay đổi giá trị
Rules of Hooks: Đừng gọi Hooks trong vòng lặp, điều kiện hoặc hàm lồng nhau
ReactDOM: Cập nhật DOM, nơi mà element thay đổi
Synthetic event: Các sự kiện như onClick, onChange, …
Virtual DOM: Thực hiện thao tác so sánh với DOM thật để rerender hiệu quả
Redux: Quản lý state thông qua store
